# 03. 타입과 추상화

## 1. 추상화 (Abstraction)

복잡한 현실 세계를 단순화하여 이해하기 위한 인간의 본능적인 시도입니다.

### 💡 핵심 원리

- **공통점 추출 (Generalization):** 구체적인 사물들 간의 공통점은 취하고 차이점은 버려 단순화합니다. (예: 트럼프 병사, 여왕 등을 '트럼프'로 묶음)
- **불필요한 정보 제거 (Simplification):** 중요한 부분을 강조하기 위해 지엽적인 세부 사항을 제거합니다. (예: 해리 백의 지하철 노선도)

### 🗂️ 개념의 구성 요소

객체를 분류하기 위한 '틀'을 의미하며, 세 가지 요소로 정의됩니다.

- **심볼(Symbol):** 개념을 가리키는 이름 (예: `트럼프`)
- **내연(Intension):** 개념의 정의. 객체가 이 개념에 속하는지 판단하는 기준 (예: `납작하고 몸이 종이로 된 존재`)
- **외연(Extension):** 개념에 속하는 실제 객체들의 집합 (예: `하트 여왕`, `병사들`)

## 2. 타입 (Type)

타입은 데이터가 어떻게 사용되는가에 대한 **추상적인 분류**입니다.

### ⚙️ 타입의 특징

1. **행동이 우선한다:** 데이터가 어떤 연산(행동)을 할 수 있느냐가 타입을 결정합니다.
    - *Duck Typing:* "오리처럼 걷고 꽥꽥거리면 그것은 오리다."
2. **데이터 표현의 은닉:** 내부적으로 데이터가 메모리에 어떻게 저장되는지는 중요하지 않습니다. 오직 외부로 노출된 **행동**만이 타입을 결정합니다.

> **결론:** 객체의 타입은 객체가 수행하는 **행동**에 의해 결정됩니다. 동일한 행동을 수행한다면 동일한 타입으로 간주합니다.

## 3. 타입의 계층: 일반화와 특수화

객체 간의 관계를 더 세분화하여 계층 구조를 만듭니다.

| 구분 | 개념 | 특징 |
| --- | --- | --- |
| **슈퍼타입 (Supertype)** | **일반적**인 타입 | 서브타입의 공통 행동을 포함 (예: 트럼프) |
| **서브타입 (Subtype)** | **특수**한 타입 | 슈퍼타입의 행동 + 자신만의 특수 행동 (예: 트럼프 인간) |

Sheets로 내보내기

- **행위 호환성:** 서브타입은 슈퍼타입의 행위를 수행할 수 있어야 합니다. (LSP 원칙의 기초)
- **다형성:** 동일한 메시지(요청)를 수신하더라도, 각 객체의 타입에 따라 다르게 반응할 수 있는 능력입니다.

## 4. 정적 모델 vs 동적 모델

### 🏃 동적 모델 (Dynamic Model)

- 프로그램이 **실행 중**일 때 객체의 상태가 어떻게 변하는지 포착하는 것.
- 스냅샷처럼 특정 시점의 객체 상태를 관찰합니다.

### 🧊 정적 모델 (Static Model)

- 상태 변화를 제거하고 객체의 타입을 **설계 시점**에서 정의하는 것.
- 코드(Class) 수준에서의 표현이며, 객체의 동적인 복잡성을 추상화합니다.

Swift

`// Alice의 상태(height)는 실시간으로 변하지만, 
// 타입(Class)은 변하지 않는 정적인 틀을 제공합니다.
class Alice {
    var height: Double // 동적 상태를 담는 변수
    
    // 행동(연산)이 타입을 정의함
    func eatMushroom() { /* 키가 커짐 */ }
    func drink()       { /* 키가 작아짐 */ }
    func fan()         { /* 키가 작아짐 */ }
}`
