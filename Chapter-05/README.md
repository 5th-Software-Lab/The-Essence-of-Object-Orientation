# 05. 책임과 메시지

## 1. 자율적인 책임

### 책임의 구체성

- 책임은 너무 구체적이면 안 된다
    - 좋은 예시 : 증언하라
    - 나쁜 예시 : 시간 순서로 사건을 설명하라
- 책임은 너무 추상적이면 안 된다
    - 좋은 예시 : 오늘 오전에 있었던 일에 대해 설명하라
    - 나쁜 예시 : 설명하라
- 자율적인 책임은 ‘어떻게’ 아닌 ‘무엇’을 할지 생각하는 것
- 책임은 어떤 행동을 수행하는 것이며, 메시지 수신 시에만 행동

## 2. 메시지와 메서드

### 메시지

- 메시지 이름과 인자로 구성
    - ex) 증언하라(어제, 왕국)

### 메서드

- 메시지 처리를 위해 객체가 선택하는 방법
- 객체지향 언어에서는 클래스 안의 함수 활용
- 실행 시간에 객체가 어떤 메서드를 사용할지 결정할 수 있음

### 다형성

- 서로 다른 유형의 객체가 동일한 메시지에 대해 서로 다르게 반응하는 능력
- 역할의 대체 가능성을 의미
- 송신자 입장에서는 어떤 수신자인지 알 필요가 없음
    - 메시지는 유연하고 확장 가능성있는 협력을 만든다

## 3. 인터페이스

### 개념

- 사물/사람 간 경계에서 상호작용할 수 있도록 도와주는 방법이나 장치

### 특성

- 사용법만 알면 내부 동작은 몰라도 된다
- 내부 구성이나 동작이 바뀌어도 인터페이스는 동일하게 작용한다
- 인터페이스가 동일하다면 어떤 대상과도 소통할 수 있다

### 인터페이스와 구현의 분리

- 객체 관점에서 생각하기
    - 좀 더 추상적인 인터페이스 : 너무 구체적이면 자율성을 해친다
    - 최소 인터페이스 : 외부에 필요없는 정보는 최대한 노출하지 않는다
    - 인터페이스와 구현 간에 차이가 있음을 인식
